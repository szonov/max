# events

Небольшой потокобезопасный пакет для работы с событиями в Go.

Пакет предоставляет простой механизм подписки и отправки событий
с поддержкой `context.Context`.

## Назначение

`events` используется для слабого связывания компонентов приложения.

Компонент может публиковать события, не зная о том, кто их получает,
а другие компоненты могут подписываться на интересующие события.

Пакет не содержит бизнес-логики и не привязан к конкретному приложению.

## Возможности

- потокобезопасная подписка на события;
- отправка событий из нескольких goroutine;
- поддержка событий без данных;
- поддержка событий с одним параметром;
- передача `context.Context` в обработчики.

## Установка

```bash
go get github.com/szonov/max/events
```

Использование:

```go
import "github.com/szonov/max/events"
```

## Пример

### Событие без данных

```go
var Ready events.Event

Ready.Subscribe(
	func(ctx context.Context) {
		fmt.Println("ready")
	},
)

Ready.Emit(ctx)
```

### Событие с данными

```go
var Message events.EventOf[string]

Message.Subscribe(
	func(ctx context.Context, value string) {
		fmt.Println(value)
	},
)

Message.Emit(ctx, "hello")
```

## Контекст

Каждый обработчик получает `context.Context`.

Это позволяет передавать:

- отмену операции;
- дедлайны;
- значения, связанные с текущей операцией.

Пример:

```go
Message.Subscribe(
	func(ctx context.Context, value string) {
		select {
		case <-ctx.Done():
			return
		default:
			process(value)
		}
	},
)
```

## Потокобезопасность

`Event` и `EventOf[T]` безопасны для использования из нескольких goroutine.

Можно подписываться и отправлять события одновременно.

## Ограничения

Пакет специально остаётся простым.

Он не предоставляет:

- очереди сообщений;
- гарантии доставки;
- хранение событий;
- отмену отдельных подписок;
- асинхронный запуск обработчиков.

Если требуется сложная маршрутизация событий,
следует использовать специализированные event bus решения.
